# Git과 Vercel의 관계 - 쉽게 이해하기

## 🎯 핵심 개념

### Git = 코드 저장소 (백업 창고)

- **Git**: 코드를 저장하고 관리하는 도구
- **GitHub**: Git을 사용하는 온라인 저장소 (구글 드라이브 같은 개념)
- 코드의 **변경 이력**을 모두 저장
- 여러 사람이 협업할 수 있게 해줌

### Vercel = 웹사이트 호스팅 (실제 사이트)

- **Vercel**: 웹사이트를 실제로 인터넷에 올려주는 서비스
- 사용자들이 접속할 수 있는 **실제 웹사이트**를 만들어줌
- `https://your-site.vercel.app` 같은 주소로 접속 가능

## 🔗 둘의 관계

```
[로컬 컴퓨터]
    ↓ (git push)
[GitHub 저장소] ← 코드 저장
    ↓ (자동 연결)
[Vercel] ← 자동으로 웹사이트 배포
    ↓
[실제 웹사이트] ← 사용자들이 보는 곳
```

## 📝 구체적인 예시

### 1. 로컬에서 작업

```
내 컴퓨터에서 코드 작성
→ 이미지 추가
→ 스타일 수정
```

### 2. Git에 저장 (백업)

```bash
git add .           # 변경사항 준비
git commit -m "..." # 변경사항 저장
git push            # GitHub에 업로드
```

### 3. Vercel이 자동으로 감지

```
GitHub에 새 코드가 올라옴
    ↓
Vercel이 자동으로 감지
    ↓
자동으로 웹사이트 다시 만들기 (재배포)
    ↓
새로운 웹사이트가 인터넷에 올라감
```

## 🎬 실제 시나리오

### 시나리오 1: 이미지 추가하기

**문제**: 로컬에서는 이미지가 보이는데, Vercel 배포 후 안 보임

**이유**:

- 로컬 컴퓨터에는 이미지가 있음 ✅
- 하지만 GitHub에는 이미지가 없음 ❌
- Vercel은 GitHub에서 코드를 가져옴
- 따라서 Vercel에도 이미지가 없음 ❌

**해결**:

```bash
# 1. 이미지를 GitHub에 올리기
git add public/images/
git commit -m "이미지 추가"
git push

# 2. Vercel이 자동으로 재배포 시작
# (별도 작업 불필요!)
```

### 시나리오 2: 코드 수정하기

1. **로컬에서 수정**

   - `components/Pricing.tsx` 파일 수정
   - 색상 변경, 버튼 추가 등

2. **Git에 저장**

   ```bash
   git add components/Pricing.tsx
   git commit -m "가격 섹션 수정"
   git push
   ```

3. **Vercel 자동 배포**
   - GitHub에 새 코드가 올라옴
   - Vercel이 자동으로 감지
   - 1-2분 후 새 웹사이트가 업데이트됨

## 🔄 자동화의 마법

### Vercel의 자동 배포 기능

```
GitHub 저장소와 Vercel을 연결하면:

1. GitHub에 코드 푸시 (git push)
   ↓
2. Vercel이 자동으로 감지
   ↓
3. 자동으로 빌드 (npm run build)
   ↓
4. 자동으로 배포
   ↓
5. 웹사이트 업데이트 완료!
```

**장점**:

- 수동으로 배포할 필요 없음
- 코드만 푸시하면 자동으로 사이트 업데이트
- 실수로 배포를 깜빡할 일이 없음

## 📊 비교표

| 항목        | Git/GitHub  | Vercel                 |
| ----------- | ----------- | ---------------------- |
| 역할        | 코드 저장소 | 웹사이트 호스팅        |
| 저장하는 것 | 소스 코드   | 실제 웹사이트          |
| 접근 방법   | GitHub.com  | your-site.vercel.app   |
| 업데이트    | git push    | 자동 (GitHub 연결 시)  |
| 목적        | 코드 관리   | 사용자에게 사이트 제공 |

## 🎓 비유로 이해하기

### Git/GitHub = 구글 드라이브

- 파일(코드)을 저장하는 곳
- 여러 버전을 관리
- 백업 역할

### Vercel = 실제 상점

- 구글 드라이브에 있는 파일을 가져와서
- 실제 상점(웹사이트)을 만드는 곳
- 사람들이 실제로 방문할 수 있음

## 💡 핵심 정리

1. **Git/GitHub**: 코드를 저장하는 곳 (백업)
2. **Vercel**: 웹사이트를 실제로 만드는 곳 (배포)
3. **연결**: GitHub에 코드를 올리면 → Vercel이 자동으로 웹사이트 업데이트
4. **문제**: GitHub에 없는 파일(이미지 등)은 Vercel에도 없음
5. **해결**: `git add` → `git commit` → `git push`로 GitHub에 올리기

## 🚀 지금 해야 할 일

이미지가 안 보이는 문제를 해결하려면:

```bash
# 1. 이미지를 Git에 추가
git add public/images/

# 2. 저장
git commit -m "이미지 추가"

# 3. GitHub에 업로드
git push

# 4. Vercel이 자동으로 재배포 (1-2분 소요)
# 5. 완료! 이미지가 보일 것입니다
```

## ❓ 자주 묻는 질문

**Q: Vercel에 직접 파일을 업로드할 수 없나요?**
A: 가능하지만 권장하지 않습니다. Git을 통해 관리하는 것이 더 안전하고 편리합니다.

**Q: git push를 안 하면 Vercel에 반영이 안 되나요?**
A: 네, 맞습니다. GitHub에 올라간 코드만 Vercel이 사용합니다.

**Q: 로컬에서는 보이는데 Vercel에서는 안 보이는 이유는?**
A: 로컬 파일이 GitHub에 올라가지 않았기 때문입니다. `git add`와 `git push`가 필요합니다.
